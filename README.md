# for 곰신(forGomsin)


## Summary
#### 주변에 군대 간 남자친구를 둔 친구들, 즉 고무신들이 많아지면서 전역일 계산에 어려움을 토로하는 사람들이 여럿이 있었기 때문에 개발을 생각하게 되었습니다.
#### 주요 기능은 날짜 계산과 다이어리 작성입니다. 사귄 날짜를 계산하여 기념일을 한눈에 알아볼 수 있고, 군인 애인의 복무 형태와 입대일을 활용하여 전역일을 계산할 수 있습니다. 그리고 간단한 다이어리를 작성하고 다시 읽어볼 수 있습니다. 또한 계급 확인이 가능하며 계급별 정보들을 재미있게 정리한 앱입니다.
#### 제가 실제 사용자라고 생각하고 사용하기 편하도록 다양한 위젯과 XML 코드를 활용해 UI에 많은 신경을 썼고, Java 코드와 SQLite를 활용하여 Backend 부분을 구현해 냈다는 것에 많은 의의가 있는 프로젝트입니다.

#### 시작화면 : 시작화면에 등록과 로그인 페이지를 생성하고 등록 후엔 등록 버튼이 보이지 않도록 설정하였습니다.
#### 정보 등록 화면 : 이름과 애인이름, 복무형태, 만난날, 입대일, 로그인에 필요한 비밀번호를 입력받습니다.
#### 시작메뉴 : 메뉴를 이용해 앱 정보와 문의할 장소를 표시합니다.
#### 메인화면 : 로그인 후 복무 형태에 따라 캐릭터가 바뀌고 메인화면에는 캐릭터와 오늘 날짜, 등록할 때의 정보를 가져와 사귄 날짜, 전역하려면 남은 날 수, 전역일이 표시됩니다.
#### 계급확인 화면 : 프로그레스바를 이용하여 버튼을 클릭하면 계급과 진행률, 군대간지 며칠이 지났는지 확인 가능합니다.
#### 캘린더 화면 : 캘린더뷰를 이용하여 날짜를 누른 뒤, 일기를 입력할 수 있는 간단한 다이어리 기능을 구현하였습니다.
#### 계급별 정보 화면 : 프래그먼트를 이용한 계급별 (이병, 일병, 상병, 병장)의 간략한 정보를 전달하도록 구현하였습니다.
#### 드로우 메뉴 :  드로우 레이아웃의 구현을 통해 콘텐츠가 한눈에 들어오고, 사용자가 어플을 좀 더 편리하게 사용할 수 있도록 메뉴를 정리해 놓았습니다.
#### 이름, 애인이름, 복무 형태 등이 적힌 등록 페이지를 제출하면 정보를 가져와 DB_join이라는 데이터 베이스에 저장이 되도록 하고, 제목, 일기내용, 하루의 평점, 날짜가 적힌 일기 쓰기 페이지를 제출하면 데이터를 가져와 DB_diary에 저장되도록 하였습니다.


## Result
#### 개발 결과 디데이와 전역 날짜 등은 LocalDB에서 날짜를 불러와 java코드로 계산하여 구현할 수 있었고, 일기 쓰기는 단순히 LocalDB를 입력하고 불러오는 것 뿐만 아니라 캘린더 뷰와 통합하여 구현하였습니다. 메뉴와 프래그먼트 등의 다양한 레이아웃은 어플의 유용함과 디자인을 더해주었다고 생각합니다.
#### 모바일 환경에서 작동하는 애플리케이션이기 때문에 레이아웃을 정하는데 어려움이 있었지만 강의 내용 외의 심화부분을 찾아 앱을 유저친화적인 모습으로 완성할 수 있었습니다. 또한 데이터베이스를 활용한 프로젝트였기 때문에 더욱 앱을 완성도 있게 마무리할 수 있었습니다.
